# Padroes de Interface SmartSenior

Diretrizes visuais para componentes de UI, baseadas na análise do moodboard oficial.

---

## Filosofia de Design

A interface SmartSenior é **acolhedora, clara e acessível**. Utilizamos formas arredondadas, sombras suaves e espaçamentos generosos para criar uma experiência confortável para usuários seniores.

> **Princípio central:** Cada elemento deve ser fácil de ver, entender e interagir.

---

## Border Radius

### Sistema de Arredondamento

O moodboard revela uma preferência por **cantos bem arredondados**, transmitindo suavidade e acessibilidade.

| Token | Valor | Uso |
|-------|-------|-----|
| `--radius-xs` | 4px | Elementos mínimos (tags inline) |
| `--radius-sm` | 6px | Inputs pequenos, chips |
| `--radius-md` | 10px | Botões, inputs padrão |
| `--radius-lg` | 14px | Cards pequenos, dropdowns |
| `--radius-xl` | 18px | Cards médios, modais |
| `--radius-2xl` | 24px | Cards grandes, seções |
| `--radius-3xl` | 32px | Hero cards, banners |
| `--radius-full` | 9999px | Avatares, badges, pills |

### Tokens Semânticos

```css
--radius-card: 20px;     /* Cards padrão */
--radius-button: 12px;   /* Botões */
--radius-input: 10px;    /* Campos de entrada */
--radius-badge: 9999px;  /* Badges (pill) */
--radius-avatar: 9999px; /* Avatares (circular) */
--radius-image: 16px;    /* Imagens em cards */
```

## Sombras

### Sistema de Elevação

Sombras no SmartSenior são **suaves e difusas**, evitando contraste agressivo.

| Token | Valor | Uso |
|-------|-------|-----|
| `--shadow-xs` | Quase imperceptível | Elementos sutis |
| `--shadow-sm` | Leve elevação | Botões, inputs |
| `--shadow-md` | Elevação média | Cards em repouso |
| `--shadow-lg` | Elevação alta | Cards em hover |
| `--shadow-xl` | Elevação máxima | Modais, drawers |

### Tokens Semânticos

```css
--shadow-card: 0 4px 20px -4px oklch(0 0 0 / 0.08);
--shadow-card-hover: 0 8px 30px -6px oklch(0 0 0 / 0.12);
--shadow-button: 0 2px 8px -2px oklch(0 0 0 / 0.10);
--shadow-dropdown: 0 10px 30px -5px oklch(0 0 0 / 0.15);
```

## Espacamentos

### Sistema de Spacing

Baseado em unidade de 4px, com espaçamentos generosos para melhor legibilidade.

| Contexto | Valor Recomendado |
|----------|------------------|
| Gap entre ícone e texto | 8px (`--spacing-2`) |
| Padding de botão | 12px 24px |
| Padding de input | 12px 16px |
| Padding de card | 20-24px |
| Gap entre cards | 16-24px |
| Margem entre seções | 32-48px |

### Áreas de Toque (Acessibilidade)

Para usuários seniores, áreas de toque devem ter **mínimo 44x44px** (WCAG 2.2).

```css
.touch-target {
  min-height: 44px;
  min-width: 44px;
}
```

## Checklist de Implementacao

Ao criar novos componentes, verifique:

- [ ] Border radius segue o sistema de tokens?
- [ ] Sombras sao suaves e difusas?
- [ ] Espacamentos sao generosos (acessibilidade)?
- [ ] Areas de toque tem minimo 44x44px?
- [ ] Cores seguem a paleta SmartSenior?
- [ ] Transicoes sao suaves (200-300ms)?
- [ ] Hover states estao definidos?
